# Signal UI — Master Implementation Plan

> **Last updated:** July 3, 2026
> **Status:** Fresh start — beginning from Phase 0

---

## 0. Project Summary

**Signal UI** is an open-source, motion-native, AI-product-ready UI component library and documentation website. It competes with shadcn/ui, 21st.dev, ReactBits, Magic UI, and Aceternity UI — differentiated by:

1. **Curated, not crowdsourced** — one design language across every component
2. **Motion-native by default** — shared duration/easing tokens, not bolted-on animations
3. **AI Product Kit** — a first-class category no competitor does well (chat threads, agent traces, tool cards)
4. **shadcn CLI-compatible** — `npx shadcn add <url>` works on day one, zero CLI code needed
5. **Open-core model** — generous MIT free tier + paid Pro pack

1. **Dark-mode-first** — deep, rich dark theme with gradient accents as default

4. **Premium feel** — glassmorphism, grain textures, glow effects, gradient borders
5. **Browsable & discoverable** — sidebar nav with counts, `Cmd+K` command palette, filterable grids

## 4. Design Token System

All tokens as CSS custom properties in `packages/ui-core/src/styles/tokens.css`. Imported globally. **Never hardcode values in components.**

### Colors (dark theme default, light theme available)

`--background`, `--foreground`, `--card`, `--card-foreground`, `--popover`, `--popover-foreground`, `--primary`, `--primary-foreground`, `--secondary`, `--secondary-foreground`, `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`, `--destructive`, `--destructive-foreground`, `--border`, `--input`, `--ring`, `--chart-1` through `--chart-5`

### Border Radius

Base: `--radius: 0.625rem` (10px). Derived: `--radius-sm`, `--radius-md`, `--radius-lg`, `--radius-xl`, `--radius-2xl`, `--radius-full`

### Motion Contract (brand-critical)

| Token | Value | Use for |
|---|---|---|
| `--motion-duration-fast` | `150ms` | Micro-interactions (button press, toggle) |
| `--motion-duration-base` | `300ms` | Layout shifts, reveals, panel open/close |
| `--motion-duration-slow` | `500ms` | Page transitions, large reveals |
| `--motion-ease-default` | `cubic-bezier(0.25, 0.1, 0.25, 1)` | Standard ease |
| `--motion-ease-out` | `cubic-bezier(0.215, 0.61, 0.355, 1)` | Entrances |
| `--motion-ease-in` | `cubic-bezier(0.55, 0.055, 0.675, 0.19)` | Exits |
| `--motion-ease-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Springy effects |

### Typography

`--font-sans` (Inter), `--font-mono` (JetBrains Mono), scale from `--text-xs` to `--text-4xl`

### Shadows

`--shadow-xs` through `--shadow-xl`

## 5. Component Anatomy (Non-Negotiable)

Every component ships as exactly:

1. **`{name}.tsx`** — the primitive
   - Uses `cva` for variants (always, even with one variant)
   - Imports `cn` from `@/lib/utils` (exact path — required for CLI install)
   - Uses Radix primitives for interactive elements
   - Adds `data-slot="{name}"` on root element
   - Full a11y: keyboard nav, focus-visible ring, ARIA attributes, disabled states
2. **`{name}.demo.tsx`** — 2-4 variant demos
   - Named exports only (no default export)
   - Each demo is standalone (no shared state)
   - Imports from `"./{name}"` (relative sibling)
3. **Registry entry** in root `registry.json`
4. **Docs page** at `apps/web/app/components/[slug]/page.tsx`

### Registry Item Format

```json
{
  "name": "button",
  "type": "registry:ui",
  "title": "Button",
  "description": "One-line description.",
  "dependencies": ["class-variance-authority", "@radix-ui/react-slot"],
  "registryDependencies": [],
  "files": [
    {
      "path": "packages/registry/new-york/ui/button.tsx",
      "type": "registry:ui"
    }
  ]
}
```

File paths relative to repo root. Use `@/` alias in component imports.

## 9. Non-Negotiable Rules

1. **Component anatomy:** `{name}.tsx` + `{name}.demo.tsx` + registry entry (§5)
2. **Imports:** `@/` alias paths only inside component files
3. **Tokens:** No hardcoded colors, durations, or radii — use CSS variables
4. **Primitives:** Radix UI for all interactive components
5. **Variants:** `cva` for every component
6. **Validation:** `shadcn build` must pass before marking done
7. **Dark mode:** Must work by default
8. **Accessibility:** Keyboard nav, focus-visible ring, ARIA, disabled states
9. **Do not hand-edit:** `public/r/` (generated), `pnpm-lock.yaml`
